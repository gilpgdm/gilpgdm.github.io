A. index.html - Página principal

  • Página principal de las aplicaciónes móviles incluye una barra de aplicación que contiene el título de la aplicación y botones de acción.

  • La página principal de la aplicación muestra el título centrado y opcionalmente, solo botones para navegación y acceso al perfil, mostrando el avatar del usuario.

  • Para que el contenido se muestra adecuadamente, se define el elemento personalizado center-aligned-top-app-bar y su comportamiento lo definimos con la clase CenterAlignedTopAppBar.

  • Para el título, dentro de la la barra de aplicación se usa un elemento de tipo h1 con el atributo slot="headline".

  • El botón de navegación opcional, debe llevar al atributo slot="navigation".

  • El botón de perfil opcional, debe llevar al atributo slot="avatar".

  • Para añadir elementos debajo del título, usa el atributo slot="adicional".

  • Hasta el final del archivo se selecciona el ancla de la varra de navegación que corresponde a esta página.

B. secudaria.html - Vista secundaria

  • Cuando la barra de aplicación no está en la página principal, puede mostrar hasta 3 botones de ícono a la derecha. Deben ser las acciones más usadas.

  • Cuando hay más de 3 acciones, coloca las menos usadas en el overflow menu, que se despliega cliqueando el botón con el ícono ⋯ en iOS, y el ícono ⁝ en otros sistemas operativos.

  • Cuando en iOS no hay íconos de acción, el título debe ir centrado.

  • Para que el contenido se muestra adecuadamente, se define el elemento personalizado top-app-bar para este tipo de páginas. Su comportamiento lo definimos con la clase TopAppBar.

  • Para el overflow menu, se define el elemento personalizado overflow-menu y su comportamiento lo definimos con la clase OverflowMenu.

  • Para el título, dentro de la la barra de aplicación se usa un elemento de tipo h1 con el atributo slot="headline". El texto debe ser corto y caber en un solo renglón, o de lo conrario se cortará.

  • Para mostrar un título más largo, hay que poner a la barra de aplicación la clase medium, agregar en esta barra, otro elemento de tipo h1 con el atributo slot="headline-small". Cuando la página tenga el scroll hasta arriba, se despliega este encabezado. Al hacer scroll, este título se compacta y se muestra el título definido con slot="headline".

  • El botón de navegación opcional, debe llevar al atributo slot="navigation".

  • Los botones de acción deben llevar al atributo slot="action".

  • El overflow menú debe llevar al atributo slot="overflow".

  • Para añadir elementos debajo del título, usa el atributo slot="adicional".

  • El botón de ícono para abrir el overflow menu, se coloca dentro de este menú usando el atributo slot="icon".

  • Los botones de acción se colocan dentro del overflow menu sin ningún atributo slot.

  • Hasta el final del archivo se selecciona el ancla de la varra de navegación que corresponde a esta página.

D. campos.html - Campos de texto

  • Se usa una etiqueta flotante que se muestra grande cuando el contenido de campo es una cadena vacía y se muestra pequeña cuando el elemento se está capturando o cuando su contenido no está vacío.

  • Para que el contenido se muestra adecuadamente, se define el elemento personalizado text-field y su comportamiento lo definimos con la clase TextField.

  • Las etiquetas deben llevar al atributo slot="label".

  • Los campos de captura deben llevar al atributo slot="target".

  • Los textos de ayuda deben llevar al atributo slot="supporting".

  • Usa class="float" cuando quieras que la etiqueta esté arriba todo el tiempo.

F. select.html - Selects

  • Este control se adapta mejor a Material Designe que el select de HTML.

  • Para que el contenido se muestra adecuadamente, se define el elemento personalizado select-menu y su comportamiento lo definimos con la clase SelectMenu.

  • Para las opciones se define el elemento personalizado menu-option y su comportamiento lo definimos con la clase MenuOption. Deben colocarse dentro de un elemento select-menu.

  • Las etiquetas deben llevar al atributo slot="label".

  • Para asignar un valor inicial, usa el atributo value.

  • Los textos de ayuda deben llevar al atributo slot="supporting".

  • Para que forzosamente se deba seleccionar una opción, agrega el atributo booleano required.

  • Para que el valor de este elemento se incluya al hacer submit, agrega el atributo name.

I. listas.html - Listas

  • Las listas tienen los siguientes elementos:

    • Un encabezado que muestra texto en negritas.

    • Opcionalmente un texto secundario que muestra mas pequeño y de ancho normal.

    • Opcionalmente muestran una imagen a la izquierda, que puede ser:

      • Un pequeño ícono cuadrado.

      • Una imagen cuadrada.

      • Una imagen rectangular.

      • El avatar de un usuario. (Se muestra dentro de un círculo.)

  • Las listas deben usar un elemento ul con la class list.

  • Los elementos de la lista se colocan en elementos li.

  • Si el elemento es un enlace, se coloca un elemento a dentro del li.

  • Si el elemento solo lleva un renglón, se coloca la class one-line dentro del elemento li. o dentro del elemento a, si es un enlace.

  • Si el elemento es de 2 renglones, se coloca la class two-line dentro del elemento li. o dentro del elemento a, si es un enlace.

  • Si el elemento es de 3 renglones, se coloca la class three-line dentro del elemento li. o dentro del elemento a, si es un enlace.

  • Todos los elementos llevan un span con la class headline. para el título, así como un elemento span con la class supporting, para el texto secundario.

  • La imágenes se pueden agregar como un material symbol o como un elemento img. El elemento debe añadir una clase para indicar el tipo de imagen que lleva, que puede ser:

    • icon.

    • image.

    • video.

    • avatar.

B. Formulario para móvil

  • Los formularios móviles muestran los mensajes de error del componente en otro componente conocido como texto de ayuda y que en este ejemplo se muestra con el id msgGenero.

  • En el resto de la lección se muestra como dar apariencia móvil a la interfaz gráfica.

<body>

  <p>
   <label for="inGenero">
    *Género
   </label>

     Pop

     Reguetón
    </option>
    <option value="tango">
     Tango
    </option>
   </select>

    <output id="msgGenero">
     *Requerido
    </output>
   </small>
  </p>
  <p>
   <button>Recomendar</button>
  </p>
  <p>
   <label for="outSalida">
    Resultado
   </label>
   <output id="outSalida">
   </output>
  </p>
 </form>
 <script>
  const REQUERIDO = "*Requerido"

  function procesa() {
   try {
    copiaMensajes()
    if (inGenero.validity.valid) {
     const genero = inGenero.value
     outSalida.value =
      recomienda(genero)
    } else {
     outSalida.value = ""
    }
   } catch (e) {
    console.error(e)
    outSalida.value = e.message
   }
  }

  /** @param {string} genero */
  function recomienda(genero) {
   if (genero === "pop") {
    return "Ariana Grande."
   } else if (genero === "reg") {
    return "Bad Bunny."
   } else {
    return "De eso no conozco."
   }
  }

  function copiaMensajes() {
   msg(inGenero, msgGenero,
    REQUERIDO)
  }

  /** Si un elemento tiene un
   * mensaje de validación, lo
   * muestra en su elemento de
   * ayuda y si no, muestra un
   * texto de ayuda.
   * @param {HTMLInputElement|
   *  HTMLTextAreaElement|
   *  HTMLSelectElement|
   *  HTMLOutputElement|null} e
   *   referencia a un elemento
   *   que contiene datos.
   * @param {HTMLOutputElement|
   *  null} out referencia a un
   *   elemento de ayuda para e.
   * @param {string} ok mensaje de
   *   ayuda cuando el estado de e
   *   es válido. */
  function msg(e, out, ok) {
   if (e && out) {
    if (e.validity.valid) {
     out.value = ok
    } else {
     out.value =
      e.validationMessage
    }
   }
  }
 </script>
</body>

D. Campos

  • Los campos de texto en Material Design están definidas en https://material.io/components/text-fields.

  • Los menús en Material Design están definidos en https://material.io/components/menus.

  • Se usa una etiqueta flotante que se muestra grande cuando el contenido de campo es una cadena vacía y se muestra pequeña cuando el elemento está en uso o cuando su contenido no está vacío.

  • Para que el contenido se muestra adecuadamente, se define el elemento personalizado cm-din y su comportamiento lo definimos con la clase CmDin localizada al final del código.

A. PWA

  • Aplicación que se entrega a través de la web, creada utilizando tecnologías web comunes como HTML, CSS y JavaScript.

  • Se conoce com PWA por sus siglas en inglés, que significan Progressive Web App.

  • Está destinado a funcionar en cualquier plataforma que use un navegador compatible con los estándares.

  • La funcionalidad incluye:

    • trabajar sin conexión,

    • notificaciones push y

    • acceso al hardware del dispositivo.

    Esto permite crear experiencias de usuario similares a las aplicaciones nativas en dispositivos móviles y de escritorio.

  • Dado que una aplicación web progresiva es un tipo de aplicación web, no hay ningún requisito para que los desarrolladores o usuarios instalen las aplicaciones web a través de sistemas de distribución digital como Apple App Store o Google Play.

  • Es posible subirlas a Google Play y a Microsoft Store.

  • Fuente: https://es.wikipedia.org/wiki/Aplicaci%C3%B3n_web_progresiva

B. Requerimientos de instalación y uso

  • Un archivo de manifiesto web, con los campos correctos completados.

  • Un servidor web utilice un dominio seguro (HTTPS).

  • Un conjunto de iconos para representar la aplicación en el dispositivo.

  • Un archivo service worker registrado para permitir que la aplicación funcione sin conexión.

  • Fuente: https://developer.mozilla.org/es/docs/Web/Progressive_web_apps/Installable_PWAs

C. Navegadores compatibles

  • Google Chrome.

  • Microsoft Edge (basado en Chromium).

  • Apple Safari (no usa el archivo de manifiesto).

D. Herramientas de desarrollo

  • Cualquier herramienta que permita editar HTML, CSS y JavaScript.

  • Un servidor web que use https.

  • Un editor de imágenes.

  • Generador de íconos enmascarables https://maskable.app/.

 console.log(
  "Intentando cargar cache:",
  CACHE)
 // Borra todos los chaches.
 const keys = await caches.keys()
 for (const key of keys) {
  await caches.delete(key)

 // Busca el contenido del cache.

  await caches.open(CACHE)

  await cache.match(evt.request,

  return response

 async function ejecuta(fetch) {

 if (resp.ok) {

 } else if (

 }

  ConnectionInterface $from,
  $msg
 ) {
  echo "Recibido: $msg\n";
  $this->valor =

   $client->send($this->valor);
  }
 }

 public function onClose(
  ConnectionInterface $con
 ) {
  $this->clientes->detach($con);
  echo "Desconectado: " .
   $con->resourceId . "\n";
 }

 public function onError(
  ConnectionInterface $con,
  \Exception $e
 ) {

   $e->getMessage() . "\n";

C. Dispositivo.ino

  • En el siguiente código, sustituye la configuración por los datos de tu red WiFi. La URL y la huella digital deben correspponder a tu proyecto de IoT.

  • Las siguientes instrucciones son para obtener la huella digital:

    1. Usa Chrome para abrir la URL del código de la lección IoT. En este caso es https://iot2.gilbertopacheco.repl.co/.

    2. Haz clic en el candadito que está junto a la URL. Se abre un menú contextual.

    3. Haz clic en donde dice Certificado (válido). Aparece un cuadro de diálogo. Observa hasta cuando puedes usar esta información. Despues de la fecha indicada, debes actualizar tu código.

    4. Selecciona la pestaña Detalles. Haz scroll hasta que aparezca la Huella digital. Haz clic en esta propiedad y abajo se muestra su valor. Copia y pega este valor a tu código.

  • Para monitorear su funcionamiento, debes hacer clic en el botón de arriba a la derecha en la ventana de Arduino, que tiene una lupa y tiene el mensaje flotante que dice Monitor Serie. Debes ajustar los baudios a la misma velocidad que en el código, donde dice Serial.begin(115200); que en este caso es 115200 baudios. Si no se ven los mensajes, baja la velocidad tanto en el código, como en el monitor.

#include <ESP8266WiFi.h>
#include <ESP8266WiFiMulti.h>
#include <ArduinoWebsockets.h>

const char* const SSID =
 "*******";
const char* const PASS =
 "*******";
const char* const HUELLA_DIGITAL =
 "*************";
const char* const URL =
 "wss://phpws.gilbertopacheco.repl.co/";

using namespace websockets;
ESP8266WiFiMulti WiFiMulti;
WebsocketsClient  client;

const int BOTON = 0;
bool presionado = false;
bool conectado = false;
String valor;

void onMessageCallback(
  WebsocketsMessage message) {
 Serial.print("Recibe ");
 valor = message.data();
 digitalWrite(LED_BUILTIN,
  valor == "1" ? LOW : HIGH);
 Serial.println(message.data());
}

void onEventsCallback(
  WebsocketsEvent event,
  String data) {
 if (event ==
   WebsocketsEvent::ConnectionOpened) {
  Serial.println("Conectado.");
  conectado = true;
 } else if (event ==
   WebsocketsEvent::ConnectionClosed) {
  Serial.println("Desconectado.");
  conectado = false;
 }
}

void setup() {
 Serial.begin(115200);
 // Serial.setDebugOutput(true);
 Serial.println();
 Serial.println();
 Serial.println();
 pinMode(LED_BUILTIN, OUTPUT);
 pinMode(BOTON, INPUT);
 for (uint8_t t = 4; t > 0;
  t--) {
  Serial.printf(
   "ESPERANDO %d...\n", t);
  Serial.flush();
  delay(1000);
 }
 Serial.println(
  "Conectando WiFi...");
 WiFi.mode(WIFI_STA);
 WiFiMulti.addAP(SSID, PASS);
 while (WiFiMulti.run() !=
   WL_CONNECTED) {
  delay(500);
 }
 Serial.println("WiFi conectado.");
 client.
   onMessage(onMessageCallback);
 client.onEvent(onEventsCallback);
 client.
   setFingerprint(HUELLA_DIGITAL);
 Serial.println("Conectando WS.");
 client.connect(URL);
}

void loop() {
 client.poll();
 if (conectado) {
  bool actual = digitalRead(BOTON);
  if (!presionado &&
    actual == LOW) {
   client.send(valor);
   Serial.print("Envía");
   Serial.println(valor);
  }
  presionado = (actual == LOW);
 }
}
